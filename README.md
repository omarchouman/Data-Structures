<h3>Big-O Notation</h3>

Big-O is the worst case scenario of the code which means the high time complexity the code gets executed in.

<h4>Big-O Cases From Best To Worst</h4>
<ul>
  <li>O(1) (Best Case)</li>
  <li>O(logn)</li>
  <li>O(n)</li>
  <li>O(n**2)</li>
  <li>O(2**n)</li>
  <li>O(n!) (Worst Case)</li>
</ul>
